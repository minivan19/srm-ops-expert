# SRM运维报告生成技能使用指南

## 任务目标
- 本 Skill 用于：基于客户运维工单数据生成稳定、高质量的 SRM 年度运维分析报告
- 能力包含：
  - 脚本：Excel 读取、数据清洗、统计计算、生成数据表格和 Markdown 框架
  - 智能体：使用**确定性模板**进行 AI 分析、标准化问题筛选、内容验证
- 触发条件：用户需要生成运维年度报告、分析运维工单数据、或生成客户服务复盘报告时

## 前置准备
- 依赖说明：scripts脚本所需的依赖包及版本
  ```
  pandas>=2.0.0
  numpy>=1.24.0
  ```

## 核心架构

### 分工模式

**脚本负责（数据处理）**：
- 读取 Excel 工单文件
- 数据清洗和预处理
- 生成统计数据和表格
- 输出 Markdown 框架（包含数据表格和占位符）

**智能体负责（AI 分析 + 质量保证）**：
- 读取脚本生成的 Markdown 框架
- **使用确定性操作手册**进行标准化 AI 分析
- **应用统一筛选标准**提取核心问题和 FAQ
- **执行内容验证机制**确保完整性
- 生成最终的完整报告

## 操作步骤

### 模式 1：直接上传 Excel 文件（推荐）

当用户上传运维工单 Excel 文件时，按以下流程执行：

#### 第一步：调用脚本生成数据框架

```bash
python3 scripts/generate_data_frame.py --file [Excel文件路径] --output-dir ./
```

**重要**：在 Skill 根目录执行脚本

参数说明：
- `--file`: Excel 工单文件路径（必需）
- `--output-dir`: 报告输出目录（可选，默认为当前目录）

输出：
- 生成 `{日期}_{客户名称}_{年份}_数据框架.md` 文件
- 包含所有统计数据、表格和 AI 分析占位符

#### 第二步：智能体进行 AI 分析（标准化流程）

**重要**：此步骤必须严格按照以下标准化流程执行，不能自由发挥

**步骤 A：准备工作**
1. 读取完整的数据框架文件
2. 阅读 [references/deterministic_analysis_manual.md](references/deterministic_analysis_manual.md)（确定性操作手册）
3. 记录所有【需要 AI 分析】占位符的位置

**步骤 B：执行 AI 分析（必须使用确定性模板）**

按照确定性操作手册的顺序执行：

**1. 第一部分：工单概览**
- 1.4 趋势分析：使用 [2.1.1 节](references/deterministic_analysis_manual.md#211-14-趋势分析确定性模板) 的确定性模板
- 1.5 SLA 与满意度分析：使用 [2.1.2 节](references/deterministic_analysis_manual.md#212-15-sla-与满意度分析确定性模板) 的确定性模板

**2. 第二部分：分类维度深度分析**
- 使用 [2.2 节](references/deterministic_analysis_manual.md#22-第二部分分类维度深度分析) 的标准化筛选标准
- 每个分类生成 3-5 个问题（严格按照工单数量确定）
- 使用标准化问题分析格式

**3. 第三部分：模块维度深度分析**
- 同第二部分的标准和格式

**4. 第四部分：高频问题 Q&A 知识库**
- 使用 [2.4 节](references/deterministic_analysis_manual.md#24-第四部分高频问题-qa-知识库) 的 FAQ 筛选标准
- 必须生成 5 个 FAQ（不能多也不能少）
- 使用标准化 FAQ 格式

**5. 第五部分：总结与建议**
- 使用 [2.5 节](references/deterministic_analysis_manual.md#25-第五部分总结与建议) 的确定性模板

**步骤 C：内容验证（强制执行）**

完成 AI 分析后，必须执行验证：

1. **占位符检查**：按照 [3.1 节](references/deterministic_analysis_manual.md#31-占位符检查清单) 的检查清单逐项验证
2. **内容质量检查**：按照 [3.2 节](references/deterministic_analysis_manual.md#32-内容质量检查) 进行质量检查
3. **验证失败处理**：如果验证失败，按照 [3.3 节](references/deterministic_analysis_manual.md#33-验证失败处理流程) 重新生成

**步骤 D：最终确认**

验证通过后，确认：
- [ ] 所有【需要 AI 分析】占位符已替换
- [ ] 所有问题有明确的原因分析
- [ ] 所有解决方案可执行、可验证
- [ ] 没有使用禁用词汇（提供、优化、加强、完善）
- [ ] 格式符合要求

#### 第三步：返回完整报告给用户

- 将填充后的完整 Markdown 报告发送给用户
- 提供关键指标摘要
- 说明报告已通过内容验证

### 模式 2：使用客户名称（固定目录场景）

当用户指定客户名称时，按以下流程执行：

1. **确保客户目录存在**
   - 客户目录位于：`/workspace/projects/workspace/documents/client/{客户名称}/`
   - 工单文件位于：`{客户目录}/运维工单/` 子目录下

   ```bash

   ```

3. **智能体进行 AI 分析并填充报告**
   - 同模式 1 的第二步（标准化流程）

4. **返回完整报告给用户**

## 关键优化点说明

### 1. 确定性模板（解决随机性问题）

**问题**：之前 AI 分析内容随机性大，每次生成结果差异大

**解决方案**：
- 所有分析步骤都有明确的模板和规则
- 趋势分析：固定的 3 步分析流程 + 标准化输出格式
- 问题分析：统一的问题格式 + 标准化解决方案结构
- 总结建议：固定的 3 部分结构 + 标准化评价体系

**预期效果**：同样的数据输入，生成的报告内容一致性 ≥ 90%

### 2. 统一筛选标准（解决结果波动问题）

**问题**：核心问题和 FAQ 的筛选标准不明确，每次选择的问题差异大

**解决方案**：
- **问题筛选量化标准**：
  - 首先按工单数量降序排列
  - 相同数量时按影响范围排序
  - 相同影响范围时按严重程度排序
- **问题数量固定规则**：
  - ≥30 单：3 个问题
  - 20-29 单：2 个问题
  - 10-19 单：1 个问题
  - <10 单：不输出
- **FAQ 筛选标准**：
  - 仅从"业务咨询"分类筛选
  - 按重复度、普适性、实用性排序
  - 必须生成 5 个（不能多也不能少）

**预期效果**：同样的数据输入，筛选出的问题重合度 ≥ 85%

### 3. 内容验证机制（解决内容缺失问题）

**问题**：AI 分析经常遗漏部分占位符，导致报告不完整

**解决方案**：
- **三级验证体系**：
  1. **占位符检查清单**：逐项检查所有占位符是否替换
  2. **内容质量检查**：检查格式、内容、禁用词汇
  3. **验证失败重跑机制**：最多重试 2 次，确保完整性
- **强制验证步骤**：
  - 验证不通过不能输出报告
  - 验证结果必须记录

**预期效果**：报告完整性达到 100%，无遗漏占位符
