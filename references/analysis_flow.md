# SRM 运维报告分析流程

本文档详细说明 SRM 运维报告生成的完整分析流程，采用"脚本数据处理 + 智能体 AI 分析"的混合架构。

## 整体架构

### 分工模式

**脚本负责（数据处理层）**：
- Excel 文件读取和解析
- 数据清洗和预处理
- 统计计算和数据聚合
- 生成数据表格和 Markdown 框架
- 输出结构化的数据供 AI 分析

**智能体负责（AI 分析层）**：
- 读取脚本生成的数据框架
- 理解数据含义和业务背景
- 进行深度 AI 分析和归纳
- 生成洞察、建议和总结
- 填充完整的报告内容

### 优势

这种混合架构的优势：
1. **发挥各自专长**：脚本擅长数据处理，智能体擅长 AI 分析
2. **无需 API 配置**：直接利用智能体自身的 LLM 能力
3. **灵活可扩展**：AI 分析部分可以根据需要调整深度
4. **用户体验好**：直接消耗扣子积分，无需额外配置

## 详细流程步骤

### 阶段一：数据处理（脚本执行）

#### 步骤 1.1：接收用户请求

**触发条件**：
- 用户上传 Excel 工单文件
- 用户指定客户名称和年份

**脚本动作**：
- 解析命令行参数
- 定位 Excel 文件路径
- 验证文件存在性和安全性

#### 步骤 1.2：读取 Excel 数据

**脚本动作**：
- 使用 pandas 读取 Excel 文件
- 处理不同的日期格式
- 识别关键字段（工单状态、分类、模块、工时等）
- 记录原始数据量

#### 步骤 1.3：数据清洗和过滤

**脚本动作**：
- 过滤已关闭工单（如果有状态字段）
- 处理缺失值
- 解析工时段字符串
- 转换日期格式

**关键统计**：
- 总工单数（仅已关闭）
- 有效工时记录数
- 时间范围

#### 步骤 1.4：统计计算

**脚本动作**：
- 计算总工单量和平均工时
- 按分类、模块、优先级统计分布
- 计算 SLA 达标率
- 计算产品满意度和服务满意度
- 计算在线提单率
- 按月统计工单趋势

#### 步骤 1.5：生成数据框架

**脚本输出**：
- Markdown 格式的数据框架文件
- 包含所有统计表格
- 包含工单样本数据
- 包含 AI 分析占位符
- 文件命名：`{日期}_{客户名称}_{年份}_数据框架.md`

### 阶段二：AI 分析（智能体执行）

#### 步骤 2.1：读取数据框架

**智能体动作**：
- 读取脚本生成的数据框架文件
- 理解所有统计数据和表格
- 查看工单样本了解业务背景
- 识别关键指标和异常点

#### 步骤 2.2：第一部分分析（工单概览）

**分析重点**：

**1.4 趋势分析**：
- 读取月度趋势数据
- 分析整体趋势特征
- 识别高峰期和低谷期
- 分析波动原因
- 根据数据特征自然分段（不强行分阶段）
- 提供趋势预测

**1.5 SLA 与满意度分析**：
- 评价 SLA 达标率表现
- 分析产品满意度和服务满意度
- 提供改进建议

#### 步骤 2.3：第二部分分析（分类维度）

**分析重点**：
- 对每个 Top 5 分类进行深度分析
- 查看该分类的工单样本
- 归纳 3-5 个核心问题
- 每个问题包含：标题、原因、解决方案、参考工单
- 提供宏观总结

#### 步骤 2.4：第三部分分析（模块维度）

**分析重点**：
- 对每个 Top 5 模块进行深度分析
- 查看该模块的工单样本
- 归纳 3-5 个核心问题
- 每个问题包含：标题、原因、解决方案、参考工单
- 提供宏观总结

#### 步骤 2.5：第四部分分析（Q&A 知识库）

**分析重点**：
- 仅分析"业务咨询"分类的工单
- 生成 5 个高频问题 FAQ
- 按频率从高到低排序
- 从最终用户角度出发
- 每个 FAQ 包含：问题、适用场景、解决方案、参考工单

#### 步骤 2.6：第五部分分析（总结与建议）

**分析重点**：

**5.1 运维整体表现**：
- 响应效率分析
- 处理效率分析
- 问题结构分析

**5.2 主要问题与风险**：
- 集成稳定性
- 培训需求
- 高峰期压力

**5.3 改进建议**：
- 短期（1-3 个月）
- 中期（3-6 个月）
- 长期（6 个月以上）

#### 步骤 2.7：生成最终报告

**智能体动作**：
- 将所有 AI 分析内容填充到数据框架
- 替换所有"【需要 AI 分析】"占位符
- 检查格式一致性
- 确保内容连贯和逻辑通顺
- 生成最终的完整 Markdown 报告

## 数据处理说明

### 字段映射

脚本会自动识别以下可能的字段名：

| 数据项 | 可能的字段名 |
|--------|-------------|
| 工单状态 | 工单状态、状态、处理状态 |
| 工单分类 | 工单分类、分类、问题类型 |
| 所属模块 | 所属模块、模块、系统模块 |
| 优先级 | 优先级、紧急程度、重要性 |
| 处理工时 | 处理工时、工时 |
| SLA 是否达标 | SLA是否达标、SLA状态 |
| 产品满意度 | 产品满意度 |
| 服务满意度 | 服务满意度 |
| 是否在线提单 | 是否在线提单、在线提单 |

### 工时解析

支持的工时格式：
- "2小时30分" → 2.5 小时
- "3小时" → 3.0 小时
- "45分" → 0.75 小时

### 极值剔除

- 数据量 ≥ 100：剔除首尾 5%
- 数据量 20-99：剔除首尾各 1 条
- 数据量 < 20：不剔除

## AI 分析要点

### 分析质量要求

1. **基于数据，不虚构**
   - 所有分析必须基于实际工单数据
   - 不要编造不存在的问题或解决方案
   - 参考工单要真实存在

2. **结构清晰，可操作**
   - 使用编号列表和清晰的标题
   - 解决方案要具体、可执行
   - 提供操作路径和注意事项

3. **深度分析，表面化**
   - 不仅描述问题，还要分析原因
   - 提供预防性建议，不仅是修复性方案
   - 有洞察和预判

### 格式规范

**问题分析格式**：
```
===问题分隔线===
问题：[简洁明了的问题标题]
原因：[分析问题产生的原因，要详细具体]
解决方案：[使用编号列表格式（1. 2. 3.），每个解决方案必须包含：具体场景描述 + 操作路径/步骤 + 注意事项]
参考工单：[相关的1-2个工单编号，用逗号分隔]
```

**FAQ 格式**：
```
===FAQ分隔线===
问题：[简洁明了的问题标题]
适用场景：[描述什么情况下会遇到这个问题]
解决方案：[清晰的步骤说明]
参考工单：[相关的1-2个工单编号]
```

### 解决方案编写要点

好的解决方案应该包含：
1. **具体场景**：在什么情况下使用
2. **操作路径**：具体的菜单路径和步骤（用【】括起模块名）
3. **注意事项**：风险提示、权限要求、生效范围等

**示例**：
```
1. 通用字段显隐、必填属性配置：进入【页面个性化】对应功能模块，编辑目标字段属性调整后保存即可，建议先在测试环境验证再配置正式环境
2. 打印模板配置：进入【单据样式定制】/【组合业务对象】对应模板路径，新增/调整所需字段后发布，新配置仅对后续新建单据生效
```
