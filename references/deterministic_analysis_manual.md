# SRM 运维报告 LLM 分析约束手册

本文档是给 **LLM 调用的提示词约束**，不是给 Agent 读的流程指南。

---

## 批次1：趋势 + SLA 分析

### 约束

1. 基于统计数据（月度分布、SLA达标率、满意度）分析
2. 趋势描述用"持续上升/先升后降/波动较大/相对平稳"
3. 满意度评价标准：≥4.5优秀，4.0-4.5良好，<4.0需改进
4. SLA评价标准：≥98%优秀，95-98%良好，<95%需改进

### 输出格式

```
### 1.4 趋势分析
**整体趋势判断** [简要描述]
**阶段划分** 高峰期/低谷期/平稳期 各月
**趋势预测与建议** 1. 2. 3.

### 1.5 SLA与满意度分析
**核心指标表现**
- SLA达标率：[数值] → [评价]
- 产品满意度：[数值]分 → [评价]
- 服务满意度：[数值]分 → [评价]
**综合分析** [根据数据选择合适场景]
```

## 批次2：分类 + 模块分析

### 约束

1. **只基于工单数据** — 不推测、不虚构
2. **解决方案来源** — 有解决方案的工单 → 提取共性；无解决方案 → 写"待完善"
3. **禁止空洞词汇** — ❌提供 ❌优化 ❌加强 ❌完善 → ✅具体操作步骤
4. **参考工单必须真实** — 只写数据中存在的工单号，禁止幻觉
5. **问题数量规则**：
   - ≥30单 → 3个问题
   - 20-29单 → 2个问题
   - 10-19单 → 1个问题
   - <10单 → 不输出

### 输出格式

```
## 第二部分：分类维度深度分析
### 2.1 [分类名] - [工单数]单，建议[数字]个问题

===问题分隔线===
问题：[简洁标题，15-25字]
原因：[2-3句话，技术原因+管理原因]
解决方案：
1. [场景描述]：[操作路径，包含【模块名】]
2. [场景描述]：[操作路径，包含【模块名】]
参考工单：[工单号，逗号分隔]

## 第三部分：模块维度深度分析
（同上格式）
```

## 批次3：FAQ 分析

### 约束

1. **筛选范围** — 只从"业务咨询"分类选，排除报错/故障类
2. **面向用户** — 是业务人员的操作指导，不是技术配置
3. **解决方案类型**：
   - ✅ 操作指导（如：点击XX→填写XX→提交）
   - ✅ 答疑解释（如：这是因为XX）
   - ✅ 联系管理员（如：请联系管理员处理）
   - ❌ 配置建议（如：进入系统设置→配置XX）
   - ❌ 优化建议（如：建议优化XX）
4. **禁止空洞词汇** — 不用"优化、完善、配置、调整"
5. **数量固定** — 必须恰好5个FAQ

### 输出格式

```
## 第四部分：高频Q&A知识库

===问题分隔线===
问题：[以"如何/怎么/什么是"开头]
适用场景：[1-2句话]
解决方案：
1. [操作步骤]
2. [操作步骤]
参考工单：[工单号]
```

## 批次4：总结与建议

### 约束

1. 基于批次2和批次3的分析结果
2. Top问题必须与前文分析一致
3. 建议分短期（1-3月）、中期（3-6月）、长期（6月+）

### 输出格式

```
## 第五部分：总结与改进建议

### 5.1 运维整体表现
**核心数据**
- 总工单：[数量]
- SLA达标率：[数值]%
- 满意度：[数值]分
**整体评价** [优秀/良好/待改进]

### 5.2 主要问题与风险识别
1. **[问题]** - [影响范围] - [风险等级]
2. **[问题]** - [影响范围] - [风险等级]
3. **[问题]** - [影响范围] - [风险等级]

### 5.3 改进建议
**短期（1-3个月）**
1. [建议]
**中期（3-6个月）**
1. [建议]
**长期（6个月以上）**
1. [建议]
```
